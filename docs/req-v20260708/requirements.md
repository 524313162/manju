# ManjuCraft.Web AI 创作能力对接 — manjucraft-ai-integration-v2 需求分析文档

> 版本：1.0 | 日期：2026-07-08 | 状态：草稿

## 1. 项目概述

### 1.1 项目背景

ManjuCraft.Web 是漫剧创作工具平台，提供项目、剧本、资产、制作分镜等核心功能模块。ComfyuiProxy.Web 代理程序已完成开发，封装了 8 种 AI 工作流（文生图、人物档案、分镜、文本生成、文生视频、图生视频、ACE 音乐、Stable BGM），所有代理接口均已调通。

当前 ManjuCraft.Web 已有基础的 CRUD 功能和手动上传资产能力，但缺少 AI 真实创作能力的深度对接。本次需求将 ComfyuiProxy.Web 的 AI 工作流能力全面接入 ManjuCraft.Web 的创作流程中。

### 1.2 项目目标

全面打通 ManjuCraft.Web 与 ComfyuiProxy.Web 的 AI 创作能力，使漫剧团队成员能够通过 Web 界面直接使用各类 AI 工具进行内容创作，实现：

- **故事创作**：通过 LLM 自动生成剧本章节和改写
- **资产生成**：AI 生成角色图、场景图、BGM 音乐，自动替换资产文件
- **分镜生成**：AI 从章节内容拆分成镜脚本，生成分镜图、首帧图、分镜视频

### 1.3 目标用户

漫剧制作团队成员（编剧、导演、美术设计等）。

### 1.4 核心价值

让 AI 创作更方便，降低人工创作门槛，提升内容生产效率。

## 3. 非功能需求

### 3.1 性能要求

| 维度          | 要求                        |
| ------------- | --------------------------- |
| 轮询间隔      | 5 秒一次                    |
| 超时时间      | 10 分钟（ComfyUI 任务等待） |
| 并发限制      | ComfyUI 不可并发，单线程执行 |
| 超时处理方式  | 返回 promptId，前端点击"获取资产"继续查询 |
| 普通大模型响应 | 同步实时返回，无轮询        |

### 3.2 安全要求

- 无用户认证和权限管理系统
- API Key 存储于 SQLite 数据库（明文）
- 文件上传无额外安全限制

### 3.3 兼容性要求

- **设备**：PC 端
- **浏览器**：主流浏览器（Chrome / Edge / Firefox）
- **系统集成**：ComfyUI 服务 + ComfyUI 代理 + ManjuCraft.Web 三个独立服务

### 3.4 可用性要求

- ComfyUI 或代理不可用时，返回错误提示

## 4. 约束与假设

### 4.1 技术约束

| 约束       | 说明                                                                                         |
| ---------- | -------------------------------------------------------------------------------------------- |
| 技术栈     | .NET 10.0，MVC，SQLite，EF Core                                                              |
| 前端       | 原生 HTML/CSS/JS，无 SPA 框架                                                                |
| 代理模式   | 所有 ComfyUI 调用统一通过 ComfyuiProxy.Web 代理                                             |
| ComfyUI    | 单线程执行，不可并发；提交后需轮询等待结果                                                   |
| Provider   | 需兼容两种 ProviderType：`ComfyUI`（异步轮询）和 `LLM`（同步实时返回）                        |
| 资产存储   | 生成后的图片/音频统一存入本地文件系统（`wwwroot/asset/{type}/{id}.{ext}`），通过 AssetsController/ReplaceResource 替换 |

### 4.2 时间约束

无明确里程碑节点。

### 4.3 假设

1. ComfyUI 服务器可用，ComfyuiProxy 代理服务器正常运行
2. ComfyUI 代理地址通过 `appsettings.json` 的 `ComfyuiProxyUrl` 配置
3. ComfyUI 工作流 JSON 文件已部署在代理服务器的 workflows 目录
4. LLM 大模型 API（DeepSeek / Qwen 等）可正常调用
5. 多任务同时触发时 ComfyUI 自动排队处理

### 4.4 风险

| 风险         | 说明                                         | 缓解措施                                     |
| ------------ | -------------------------------------------- | -------------------------------------------- |
| ComfyUI 不可用 | 服务器宕机或代理不可达                       | 前端显示明确的错误提示                         |
| 超时时间长   | ComfyUI 生图可能超过 10 分钟                 | 超时返回 promptId，前端可手动点击"获取资产"   |
| 单线程瓶颈   | ComfyUI 一次只能执行一个工作流               | 任务排队处理，前端提示正在生成中              |
| API 调用异常 | 网络问题或 ComfyUI 返回异常                  | 重试机制（代理已有）+ 友好的错误提示          |
