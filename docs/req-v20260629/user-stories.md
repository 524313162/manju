## Epic 1：项目管理

### US-001：创建漫剧项目

**优先级**：Must Have

作为漫剧创作者，
我希望创建一个新的漫剧项目，
以便开始组织我的创作内容。

**验收标准**：

- [ ] 用户可以输入项目名称创建项目
- [ ] 项目创建成功后自动跳转到项目详情页面
- [ ] 项目名称具有唯一性约束

### US-002：管理漫剧项目列表

**优先级**：Must Have

作为漫剧创作者，
我希望查看、编辑和删除已完成或草稿项目，
以便管理多个漫剧作品。

**验收标准**：

- [ ] 首页显示所有项目列表
- [ ] 用户可以编辑项目名称
- [ ] 用户可以删除项目（含级联删除关联数据）

## Epic 2：剧本创作

### US-003：主题式 AI 剧本创作（对话式）

**优先级**：Must Have

作为漫剧创作者，
我希望输入一个主题后，系统按章节与 AI 逐步对话生成剧本，
以便快速构建剧本内容。

**验收标准**：

- [ ] 用户先输入剧本标题
- [ ] 用户输入一个简短主题开始创作
- [ ] 系统逐章与用户对话，每次创建一条章节记录：`StoryChapter.ChapterNumber`、`StoryChapter.ChapterName`、`StoryChapter.Content`
- [ ] 每生成一章后，系统等待用户反馈（调整/同意/继续下一章）
- [ ] 用户可编辑已创建的章节内容（标题、名字、内容）
- [ ] 创建新剧本前，若项目中已有资产数据（角色/道具/场景/BGM/分集/分镜），弹出警告提示
- [ ] 创建剧本默认使用 DeepSeek 接口
- [ ] 章节按 `Order` 字段排序显示

### US-004：手动创建章节剧本

**优先级**：Must Have

作为漫剧创作者，
我希望手动一条一条添加章节（标题+章节名字+内容），
以便我已有完整剧本时可以逐章录入。

**验收标准**：

- [ ] 用户可以添加章节（章节序号+章节名字+章节内容），支持重新排序
- [ ] 用户可编辑章节内容
- [ ] 用户可删除章节
- [ ] 内容保存后可用于按章节生成漫剧清单

---

## Epic 3：漫剧清单生成

### US-005：从剧本章节生成漫剧清单

**优先级**：Must Have

作为漫剧创作者，
我希望选择剧本的某一章后，系统自动生成角色、道具、场景、BGM、镜头五份清单，
以便为后续素材生成做准备。

**验收标准**：

- [ ] 系统展示章节列表（章节序号+章节名字），用户选择一个章节触发
- [ ] 系统 AI 分析选中章节的 `Content`，自动生成五份清单：
  - **角色清单**：角色只生成一套基础档案（姓名+身高+年龄+外貌详细描述等），后期可在角色下添加换装要求
  - **道具清单**：道具名称+道具基本信息详细描述
  - **场景清单**：场景名称+场景基本信息详细描述
  - **BGM清单**：BGM名称+BGM描述
  - **镜头清单**：镜头号码/景别/运镜/镜头分析/时长/所需资源（角色&场景&道具引用）/首帧图片描述/视频生成提示词
- [ ] 生成的清单可手动编辑和调整
- [ ] 在角色/道具/场景等清单页面，可查看**所有章节**产生的全部对应清单对象（不局限于当前选中的章节）

## Epic 4：演员清单管理

### US-006：管理演员清单（含子角色/换装）

**优先级**：Must Have

作为漫剧创作者，
我希望管理演员（角色）基础信息，并为一个角色添加多个换装变体（子角色），
以便角色在剧本不同场景中可以穿着不同服装。

**验收标准**：

- [ ] 可以添加角色（基础档案：姓名+身高+年龄等+外貌详细描述）
- [ ] 可以编辑角色信息
- [ ] 可以删除角色
- [ ] 列表显示所有角色及基本信息
- [ ] **子角色/换装**：在角色详情页可添加多个换装变体（换装需求本质是子角色），每个子角色记录换装描述
- [ ] 角色清单页面的"全部角色"视图汇总所有章节产生的角色

### US-007：为角色生成/重新生成提示词

**优先级**：Must Have

作为漫剧创作者，
我希望使用AI根据角色描述生成角色档案提示词（包含四视图绘图布局和格式控制），也可以随时重新生成，
以便获得更好的角色四视图生成效果。

**验收标准**：

- [ ] 提示词模板存储在数据库中（PromptTemplate 实体）
- [ ] 首次生成可手动填写角色描述触发AI，或由漫剧清单自动生成
- [ ] 点击"重新生成"按钮可调AI重新生成角色提示词
- [ ] 重新生成时弹出接口选择器，可选择使用哪个LLM接口
- [ ] 支持流式输出显示生成过程
- [ ] 用户可以手动修改提示词

### US-008：为角色生成/替换图片

**优先级**：Must Have

作为漫剧创作者，
我希望一键将角色提示词发送给ComfyUI生成角色四视图图片，也可以上传自定义图片替换，
以便获得角色立绘素材。

**验收标准**：

- [ ] 每个角色旁边有"生成图片"按钮，使用ComfyUI生成角色四视图
- [ ] 点击图片区域可触发文件选择上传自定义图片
- [ ] 支持常见图片格式上传（png/jpg）
- [ ] 上传后图片立即替换显示新图

## Epic 11：API接口管理

### US-022：管理API接口配置

**优先级**：Must Have

作为漫剧创作者，
我希望在设置页中统一管理所有可用的API接口（DeepSeek/NVIDIA/Ollama的LLM类型 + ComfyUI接口），
以便在不同步骤灵活切换AI服务。

**验收标准**：

- [ ] 用户可以添加/编辑/删除接口配置
- [ ] 接口分为两种类型：LLM（文本生成）和ComfyUI（图片/视频/音频生成），统一在一个列表管理
- [ ] LLM类型支持配置：接口名称、提供商类型（DeepSeek/NVIDIA/Ollama/自定义）、API地址、API Key、模型名称
- [ ] ComfyUI类型支持配置：接口名称、API地址、API Key（可选）、默认工作流配置
- [ ] 可设置每个类型的默认接口
- [ ] 测试接口连通性功能

### US-023：选择每个功能步骤使用的接口

**优先级**：Must Have

作为漫剧创作者，
我希望在每个功能步骤中可以选择使用哪个API接口，
以便根据需求和质量调整AI服务。

**验收标准**：

- [ ] UI上每个步骤旁显示当前使用的接口标识
- [ ] **文字操作（剧本创作/清单生成/提示词生成）**：默认DeepSeek，点击可切换LLM接口
- [ ] **图片/视频/音频生成**：固定使用ComfyUI接口
- [ ] 重新生成提示词时（如角色/道具/场景/BGM），弹出界面让用户选择LLM接口

## Epic 15：数据安全

### US-030：剧本创建前资产检测与警告

**优先级**：Must Have

作为漫剧创作者，
我希望在创建新剧本前，如果项目已有资产数据（角色/道具/场景/BGM/分集/分镜等），系统弹出警告提示，
以免误覆盖已有工作成果。

**验收标准**：

- [ ] 创建新剧本时检测项目中是否存在已有资产记录

- [ ] 用户取消后不改变原有内容
