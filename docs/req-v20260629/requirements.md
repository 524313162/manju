# 漫剧开发工具 需求分析文档

> 版本：1.0 | 日期：2026-06-29 | 状态：确认

## 1. 项目概述

### 1.1 项目背景

本地有条件运行 ComfyUI 的用户，需要使用自动化工具管理漫剧制作全流程。目的是降低 API 费用成本，同时解决手动管理资产（演员、场景、道具、分镜等）分散混乱的问题。

### 1.2 项目目标

提供一站式漫剧开发工具，从故事创意开始，自动拆分和管理资产，对接 ComfyUI 生成图片/视频素材，最终合并导出完整漫剧视频。

### 1.3 目标用户

使用 ComfyUI 制作漫剧的个人或小团队，本地 PC 端操作。

### 1.4 核心价值

让漫剧制作更简单、更省时间，通过自动化工具提升效率，降低制作成本和门槛。

## 2. 功能需求

### 2.1 Must Have（必须有）

| 功能 | 描述 | 使用者 | 备注 |
|------|------|--------|------|
| 项目管理 | 创建、编辑、删除、查看漫剧项目 | 用户 | 项目名称作为唯一标识 |
| 故事管理 | 创建、编辑、删除、查看故事 | 用户 | 项目的基础输入数据 |
| 演员清单 | CRUD，含演员名、详情、四视图提示词生成 | 用户 | 四视图图片可外部上传替换 |
| 道具清单 | CRUD，含道具名、双视图提示词生成 | 用户 | 双视图图片可外部上传替换 |
| 场景清单 | CRUD，含场景名、场景图提示词生成 | 用户 | 场景图可外部上传替换 |
| 技能清单 | CRUD（可选模块） | 用户 | 按需添加 |
| BGM清单 | CRUD（可选模块） | 用户 | 按需添加 |
| 分集管理 | CRUD，含分集名称、时长 | 用户 | 每集包含多个分镜 |
| 分镜管理 | CRUD，含首帧提示词、台词、镜头提示词；增删改；拖拽排序 | 用户 | 首帧图和视频可外部上传替换 |
| 提示词自动生成 | 根据演员/道具/场景/技能/BGM名称自动生成对应提示词 | 用户 | 减少手动输入 |
| ComfyUI 接口配置 | 配置和管理 ComfyUI API 地址 | 用户 | 地址可配置，支持本地/远程 |
| 图片生成 | 调用 ComfyUI 生成演员四视图、道具双视图、场景图、首帧图 | 用户 | 生成后所有图片可外部替换 |
| 视频生成 | 调用 ComfyUI 用首帧图+提示词生成视频 | 用户 | 生成后可外部替换 |
| 视频合并导出 | 调用 FFmpeg 合并所有视频片段为完整成品 | 用户 | 合并后导出 |

### 2.2 Should Have（应该有）

| 功能 | 描述 | 使用者 | 备注 |
|------|------|--------|------|
| 预留云端 API 接口 | 架构层面预留接口，支持未来接入第三方 API | 开发者 | 本期仅架构设计，不实现对接 |

## 3. 非功能需求

### 3.1 性能要求

- 用户操作后页面显示加载状态（转圈圈）
- 资源生成完成后等待页面返回结果
- 提供获取资源按钮，用户可主动点击查看生成结果
- 防止页面刷新时丢失生成状态

### 3.2 安全要求

- 无需用户认证
- 纯本地使用，不涉及用户隐私数据保护

### 3.3 兼容性要求

- Blazor Server 模式
- 仅支持 PC 端浏览器访问

### 3.4 可用性要求

- 系统稳定运行，资源存储路径：`wwwroot/[项目号]/asset/`

## 4. 约束与假设

### 4.1 技术约束

- 前端：Blazor Server
- 后端：.NET 8 + EF Core
- 数据库：SQLite
- AI 生成：ComfyUI（地址可配置，支持本地/远程）
- 视频合并：FFmpeg 依赖本地已安装（启动前验证可用性）

### 4.2 时间约束

- 无明确上线时间要求

### 4.3 假设

- 用户本地已安装并配置好 ComfyUI
- 用户本地已安装 FFmpeg 并配置环境变量
- 用户本地磁盘空间充足
- 用户浏览器为现代 PC 浏览器

### 4.4 风险

- ComfyUI API 返回格式可能变化，需要后期适配
- 本地硬件性能影响 ComfyUI 生成速度
- 视频生成耗时较长，需做好长任务处理
