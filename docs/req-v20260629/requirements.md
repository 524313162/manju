# 漫剧开发工具 需求分析文档

> 版本：2.0 | 日期：2026-07-02 | 状态：进行中

## 1. 项目概述

### 1.1 项目背景

本地有条件运行 ComfyUI 的用户，需要使用自动化工具管理漫剧制作全流程。目的是降低 API 费用成本，同时解决手动管理资产（演员、场景、道具、分镜等）分散混乱的问题。

### 1.2 项目目标

提供一站式漫剧开发工具，从剧本创作开始，自动生成角色/道具/场景/镜头等清单，对接 ComfyUI 生成图片/视频素材，最终合并导出完整漫剧视频。

### 1.3 目标用户

使用 ComfyUI 制作漫剧的个人或小团队，本地 PC 端操作。

### 1.4 核心价值

让漫剧制作更简单、更省时间，通过自动化工具提升效率，降低制作成本和门槛。

## 2. 功能需求

### 2.1 Must Have（必须有）

| 功能 | 描述 | 使用者 | 备注 |
|------|------|--------|------|
| 项目管理 | 创建、编辑、删除、查看漫剧项目 | 用户 | 项目名称作为唯一标识 |
| 剧本创作 | 剧本标题（`Story.Title`）+ 章节列表（`StoryChapter` 每条数据一行），支持基于主题AI对话逐章生成，也支持手动填写完整剧本 | 用户 | 标题和章节各有名字+内容，逐章存入数据库；对话创作也是一章一章逐步创建
| 漫剧清单生成 | 从剧本某一章节自动生成角色/道具/场景/BGM/镜头五份清单 | 用户 | 在角色/道具/场景清单页面可查看所有章节产生的全部清单 |
| 演员清单 | CRUD，含演员基础信息（姓名+身高+年龄等）和详细描述 | 用户 | 每个角色可单独**重新生成提示词**（支持切换API接口），可**立即生成**角色四视图图片；**子角色**功能：一个角色可添加多个换装变体（换装需求本质是子角色）；四视图图片可外部上传替换 |
| 道具清单 | CRUD，含道具名称和详细信息 | 用户 | 每个道具可单独**重新生成提示词**（支持切换API接口），可**立即生成**道具图示；双视图图片可外部上传替换 |
| 场景清单 | CRUD，含场景名称和详细信息 | 用户 | 每个场景可单独**重新生成提示词**（支持切换API接口），可**立即生成**场景图示；场景图可外部上传替换 |
| 技能清单 | CRUD（可选模块） | 用户 | 按需添加 |
| BGM清单 | CRUD，含BGM名称和描述 | 用户 | 每个BGM可单独**重新生成提示词**（支持切换API接口），支持用ComfyUI MusicGen生成音频 |
| 分集管理 | CRUD，含分集名称、时长 | 用户 | 每集包含多个分镜 |
| 分镜管理 | CRUD，含：镜头号码、景别（特写/近景/中景/远景）、运镜（固定/前推/摇晃/跟随/拉远/平移）、镜头分析、时长、所需资源（角色&场景&道具引用）、首帧图片描述、视频生成提示词、台词 | 用户 | 首帧图和视频可外部上传替换；支持拖拽排序 |
| 提示词模板管理 | 可配置的提示词模板，存储在数据库中（PromptTemplate 实体） | 用户 | 包括：角色档案模板、道具档案模板、场景档案模板、BGM档案模板、剧本创作模板、漫剧清单生成模板、镜头规划模板；替代硬编码，支持运行时编辑 |
| 多接口API管理 | 在设置页管理所有可用API接口（DeepSeek/NVIDIA/Ollama/ComfyUI等），每个功能步骤可选择指定接口 | 用户 | LLM类型（文本生成）和ComfyUI类型（图片/视频/音频生成）统一在一个列表管理；**文字操作默认DeepSeek**，**图片/视频/音频生成固定ComfyUI** |
| 图片生成 | 调用 ComfyUI 生成演员四视图、道具双视图、场景图、首帧图 | 用户 | 所有生成图片的地方都用ComfyUI，生成后所有图片可外部替换 |
| 视频生成 | 调用 ComfyUI 用首帧图+提示词生成视频 | 用户 | 所有生成视频的地方都用ComfyUI，生成后可外部替换 |
| 视频合并导出 | 调用 FFmpeg 合并所有视频片段为完整成品 | 用户 | 合并后导出 |
| 剧本创建资产检测 | 创建/重新创建剧本前，检测项目中是否已有角色/道具/场景等资产数据，如有则弹出警告 | 用户 | 防止误覆盖已有工作成果，需用户确认后才能继续 |

## 3. 非功能需求

### 3.1 性能要求

- 用户操作后页面显示加载状态（转圈圈）
- 资源生成完成后等待页面返回结果
- 提供获取资源按钮，用户可主动点击查看生成结果
- 防止页面刷新时丢失生成状态
- 流式对话内容实时显示，降低用户等待焦虑

### 3.2 安全要求

- 无需用户认证
- 纯本地使用，不涉及用户隐私数据保护

### 3.3 兼容性要求

- Blazor Server 模式
- 仅支持 PC 端浏览器访问

### 3.4 可用性要求

- 系统稳定运行，资源存储路径：`wwwroot/[项目号]/asset/`
- 提示词模板可独立修改，不影响核心功能

## 4. 约束与假设

### 4.1 技术约束

- 前端：Blazor Server（或 Razor Pages/MVC）
- 后端：.NET 10 + EF Core 10
- 数据库：SQLite（含 PromptTemplate 提示词模板表）
- 剧本实体：`Story`（标题）+ `StoryChapter`（章节列表，每条含章节序号+章节名字+内容）
- AI 文本生成：可配置 DeepSeek / NVIDIA / Ollama 等 OpenAI 兼容接口，统一抽象层
- AI 图片/视频/音频生成：ComfyUI（地址可配置，支持本地/远程）
- 视频合并：FFmpeg 依赖本地已安装（启动前验证可用性）
- 提示词模板存放于 `wwwroot/templates/prompts/` 目录下，Markdown 格式

### 4.2 时间约束

- 无明确上线时间要求

### 4.3 假设

- 用户本地已安装并配置好 ComfyUI
- 用户本地已安装 FFmpeg 并配置环境变量
- 用户拥有可用的 LLM API 访问权限
- 用户本地磁盘空间充足
- 用户浏览器为现代 PC 浏览器

### 4.4 风险

- ComfyUI API 返回格式可能变化，需要后期适配
- 本地硬件性能影响 ComfyUI 生成速度
- 视频生成耗时较长，需做好长任务处理
- LLM API 接口类型多样，需要良好的抽象层避免耦合
- 剧本创作时若已有资产数据，覆盖操作需谨慎处理
