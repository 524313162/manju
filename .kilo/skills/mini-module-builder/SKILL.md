# 应用小型模块开发技能 Mini Module Builder
## 一、技能总览
本技能用于快速开发应用轻量化业务模块，全程使用中文交互输出，强制代码规范，必须完成Playwright自动化校验才算开发结束。

## 二、触发使用场景
1. 需要新建页面、接口、数据模型小型业务模块
2. 要求代码规范：一个文件仅允许定义1个类，禁止多类同文件
3. 所有代码文件、类、方法必须附带完整中文注释
4. 模块开发完成后需要自动化E2E验证，验证通过才算交付完成

## 三、强制执行流程（严格按顺序执行，不可跳过）
### 步骤1：收集模块中文业务信息（全程中文交互）
向用户询问以下必填信息，全部使用中文提问、中文提示：
1. 模块英文标识（文件夹/类名，仅英文下划线）
2. 模块中文业务名称
3. 模块完整功能描述（用于自动生成注释）
4. 模块类型选择：前端页面 / 后端接口 / 数据模型

### 步骤2：生成标准化模块代码，强制两条编码规则
1. **单文件单类约束**：拆分所有业务逻辑，每个Class独立生成单独文件，绝不允许一个文件存在多个类定义
2. **全代码注释约束**：
   - 每个文件顶部添加文件功能说明块注释
   - 每个类头部添加业务用途注释
   - 所有构造函数、业务方法添加功能、入参、返回值中文注释
3. 代码分层输出目录：model/controller/view分类存放，代码内注释全部简体中文

### 步骤3：代码规范自检，不通过则终止流程，中文报错
自动扫描全部生成代码文件，校验两条硬规则：
1. 遍历每个JS/TS文件，统计class关键字数量，任意文件class数量>1，输出中文报错：`XX文件违规，单个文件存在多个类，违反开发规范，请重新生成模块`
2. 校验文件头部是否存在标准/** 文件注释，缺失则输出中文报错：`XX文件缺少头部标准业务注释，不符合规范`
校验失败：返回步骤1，重新录入模块信息；校验通过进入下一步。

### 步骤4：自动生成Playwright自动化测试脚本
根据用户选择的模块类型自动生成对应测试代码：
1. 前端页面模块：页面路由访问、组件渲染校验、表单提交、数据回显E2E脚本
2. 后端接口模块：接口请求、状态码校验、返回字段数据校验脚本
3. 数据模型模块：增删改查数据库操作校验脚本
测试日志全部使用中文输出，记录每一步执行状态。

### 步骤5：执行Playwright自动化验证，校验通过才算技能结束
1. 运行自动生成的Playwright测试用例，捕获页面404、接口500、数据不匹配、组件加载失败等全部异常
2. 分支判定：
   - 分支A（全部用例通过）：输出中文成功提示：`模块开发完成，Playwright自动化校验全部通过，技能流程结束`，技能终止
   - 分支B（存在任意用例失败）：输出完整中文错误清单，流程终止，提示用户调整模块参数后重新执行本技能

## 四、全局硬性约束（全程必须遵守）
1. 所有与用户交互文字、报错、日志、注释全部使用简体中文，禁止英文提示
2. 任何新建代码文件，仅允许定义1个类，禁止多类共存同一文件
3. 所有生成代码必须附带完整中文注释，无注释代码视为不合格
4. 未完成Playwright自动化校验，不可判定模块开发完成，技能不能结束
5. 输出代码分层清晰，目录隔离，不堆砌所有代码至单一文件

## 五、交付输出标准
最终交付内容：
1. 分层目录的全套带注释源码（单文件单类）
2. Playwright完整自动化测试脚本
3. 中文自动化校验报告，包含用例总数、成功数、失败详情、最终结论
